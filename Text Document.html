Руководство программиста

  1. Для запуска серверой части программы необходимо наличие .NET 7. Помимо этого, необходимо подключение к базе данных neo4j, которое прописывается в файле appsetting.json.
  2. Для запуска фронт энд части необходимо установить зависимости клиентского приложения. Перейдите в папку client_app и установите зависимости проекта командой nmp install или yarn install.
  3. Запустите клиентское приложение командой npm start или yarn start.
  4. Запустите серверное приложение командой dotnet run или с помощью Visual Studio откройте проект и запустите его.
  5. Регистрация всех сервисов происходит в файле Program.cs
  6. В разработанном проекте существует сервис GraphHealthCheck.cs, который позволяет проверить работоспособность база данных. После запуска серверной части проекта зайдите по пути /healthcheck
  7. Фукнция для регистрации сервисов для базы данных регистрируются непосредственно в проекте DbManager. Данная функция вызывается в Program.cs
  8. IGeneratorRepository является базовым классов для репозиториев, если вы желаете создать свой репозиторий для конкретного узла, то в реализуйте данный интерфейс в своем репозитории
  9. Все репозитории необходимо получать через фабрику (IRepositoryFactory).
  10. Если вы желаете создать свои узлы и связи, то вам необходимо реализовать для них соответствующие интерфейсы (INode, IRelation)
